Before You Begin

If you want to use Coldfire debugger, you will require a m68k toolchain before build and debug any Coldfire related project. The debugger provides integration with GNU m68k toolchain, including make, gcc compiler, gdb debugger and Coldfire programmer. The default setting for Coldfire project builder needs GNU m68k-coff toolchain to make an coff object file capable to download into target.

The Coldfire debugger designed to work under Linux and Windows but you can change some minors to make it compatible with other platforms.


Windows

For windows the main choice is MinGW (Cygwin is also supported). MinGW is a port of GNU toolchain to the Windows platform. The m68k-coff toolchain is provided by the Coldfire debugger plugin. MinGW provides the best integration with this plugin due to its great compability with the CDT.

The following are MinGW installation instructions. Notice that the links may change over time as new versions come out.

  1. Download and run the MinGW setup program, ( MinGW-5.1.3.exe).
  2. You may select the Current or Candidate version of these tools.
  3. The MinGW setup program currently does not install the gdb bdm debugger. Thus the Coldfire debugger provides m68k-coff-gdb inside its archive. If you want different format of image files (e.g. ELF format), change the program directory from the Coldfire Debugger preferences page. The exact steps to change this path is provided in the reference section.
